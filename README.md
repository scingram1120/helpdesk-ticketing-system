# Help Desk Ticketing System Simulation

This project simulates a basic help desk ticketing system using PostgreSQL. It demonstrates how IT support issues can be tracked, documented, and resolved using a relational database structure.

## Project Overview

The database is designed to model a simple help desk workflow with three main components:

- **users**: stores employee information
- **tickets**: stores support requests and ticket status
- **ticket_updates**: stores troubleshooting steps and resolution notes

## Scenarios Included

This project includes sample tickets for common help desk issues:

- Wi-Fi connectivity problem
- Slow computer performance
- Password reset and account lockout
- Email synchronization issue
- Printer offline issue

## Skills Demonstrated

- SQL and PostgreSQL
- Relational database design
- Primary keys and foreign keys
- Ticket lifecycle tracking
- Troubleshooting documentation

## Files

- `schema.sql` – creates the database tables
- `sample_data.sql` – inserts sample users, tickets, and ticket updates
- `queries.sql` – example queries to view ticket history and reporting data

## Example Workflow

A user submits a ticket for a Wi-Fi issue. The help desk documents the issue, records troubleshooting steps, resolves the problem, and updates the ticket status to resolved. This same structure is used across multiple common support scenarios.
