# helpdesk-ticketing-system

🔹 Title

Help Desk Ticketing System Simulation

⸻

🔹 Overview

This project simulates a real-world help desk ticketing system using PostgreSQL. It demonstrates how IT support issues are tracked, documented, and resolved through structured ticket workflows.

⸻

🔹 What This Project Demonstrates
	•	Ticket lifecycle management (Open → Resolved)
	•	Troubleshooting documentation
	•	Relational database design (users, tickets, updates)
	•	Common IT support scenarios:
	•	Network connectivity (Wi-Fi)
	•	System performance issues
	•	Account lockouts/password resets
	•	Email synchronization issues
	•	Printer troubleshooting

⸻

🔹 Database Structure

	•	users → stores employee information
	•	tickets → stores issue details
	•	ticket_updates → stores troubleshooting steps

⸻

🔹 Example Ticket Workflow


A user reported they could not connect to Wi-Fi. The issue was diagnosed as a local network interruption. The device and router were restarted, restoring connectivity. The ticket was documented and marked as resolved.

⸻

🔹 Skills Used
	•	SQL (PostgreSQL)
	•	Troubleshooting methodology
	•	IT documentation
	•	Problem-solving and issue isolation
